# 推广问题诊断框架
# Agent 在制定方案前，必须先完成诊断。不诊断就给方案，等于不看病就开药。

---

## 核心诊断模型：ABCD 四象限

每家店的推广问题，本质上只有四种根源。找对根源，才能选对策略。

```
A — Awareness   知名度问题：附近的人根本不知道有这家店
B — Belief      信任度问题：知道了但不相信/不感兴趣
C — Conversion  转化问题：  感兴趣了但没有行动（没来、没下单）
D — Devotion    复购问题：  来过了但没有再来
```

---

## 如何判断这家店属于哪个象限

### 判断 A（知名度不足）的信号：
- 店开业不到 6 个月
- 没有做过任何主动推广
- 老板说"感觉没人知道我的店"
- 进店客人大多数是路过偶然发现的
- 搜索平台（小红书/大众点评）搜不到这家店

→ 策略重心：**曝光优先**。让附近的人先知道有这家店，哪里人多往哪走。

---

### 判断 B（信任度不足）的信号：
- 店已开 6 个月以上，周边人知道这家店
- 小红书/点评有一些流量但转化很差
- 顾客会进来看看但不买，或者问了很多问题但离开
- 负面评价或无评价（零评价也是信任危机）
- 老板说"好多人问但很少来"

→ 策略重心：**口碑建设**。用真实评价、真实顾客故事、产品细节内容建立信任。

---

### 判断 C（转化不足）的信号：
- 有不少人看到广告或内容（高曝光）
- 但实际到店/下单的人少
- 用户在平台上收藏了但没有来
- 老板说"广告有人看但没人来"
- 有询问但最终没有成交

→ 策略重心：**降低行动门槛**。在"看到"和"来了"之间加一个低风险的钩子（优惠、试吃、限时活动）。

---

### 判断 D（复购不足）的信号：
- 新客多，回头客少
- 进店购买的人大多只来一次
- 没有维护老顾客的机制（没有群、没有会员、没有回访）
- 老板说"感觉老客人越来越少"
- 节假日没有针对老顾客的活动

→ 策略重心：**私域激活**。建微信群、设计回购理由、用特权感维系老顾客。

---

## 多象限并存的处理原则

大多数店铺不是单一问题，但**永远只打一个主战场**：

```
新店（0-3个月）     → 优先 A，不要同时做 B/C/D
成长期（3-12个月）  → 主攻 B+C，A 维持就行
稳定期（1年以上）   → 主攻 D，同时做季节性 A
```

如果老板预算有限（<2000元/月），更要只做一件事做透，不要分散。

## 诊断提问清单（Agent 建档时必问）

以下问题用于完成诊断，每次建档时根据对话自然引入，不要像填表格：

**第一组：判断知名度**
- 「你觉得附近的人，大概有多少比例知道有你这家店？」
- 「有人路过但没进来过吗，还是说进来的人不多？」
- 「你的店在小红书/大众点评上搜得到吗？」

**第二组：判断信任度**
- 「有没有人来问了很多问题但最后没买？」
- 「你的评价/口碑现在怎么样，有顾客主动给你发过好评吗？」

**第三组：判断转化**
- 「你有没有什么优惠活动，让第一次来的人感觉'值得来一趟'？」
- 「有没有人在网上看到你但一直没来——你知道他们为什么没来吗？」

**第四组：判断复购**
- 「你有没有办法联系到以前来过的顾客？比如微信群或者备注？」
- 「老顾客多久来一次，你觉得正常吗？」

---

## 诊断完成后的输出格式（给用户呈现）

诊断完成后，要给老板一个清晰的"病情诊断"，用白话说出来：

```
模板：
「好，我大概听明白了。你现在主要的问题是[A/B/C/D白话描述]，
不是[排除掉的那个问题]的问题。
所以我们不需要[排除掉的策略]，把钱和精力放在[重点策略]上。
这个月，最值得先做的一件事是[最高优先级行动]。」
```

示例（A类店铺）：
「好，我大概听明白了。你现在主要是知名度的问题——附近的人还不知道有你这家店，所以不是产品不好，是没人发现你。
我们不需要现在就做优惠活动，把这个月的钱先用来让更多人知道你在这儿。
最值得先做的一件事是：在小红书发 5 条带地名标签的内容，让附近的人搜得到你。」
