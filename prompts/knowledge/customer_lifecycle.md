# 顾客生命周期管理：从陌生人到死忠粉的完整路径
# Agent 帮用户设计推广时，必须明确「现在要转化哪个阶段的顾客」

---

## 核心认知：不同阶段的顾客，需要完全不同的方式

一家小店的顾客，可以分成 5 个阶段。每个阶段的顾客，心理状态不同，需要的东西不同，你对他们做的事也要完全不同。

**最常见的错误：用同一种内容/活动，试图同时吸引所有阶段的人。**

```
阶段 1【陌生人】     ——从来没听说过这家店
阶段 2【好奇者】     ——看到过，有一点兴趣，但还没来
阶段 3【初访者】     ——来过一次，还不确定会不会再来
阶段 4【常客】       ——定期来，已经建立习惯
阶段 5【推荐者】     ——主动告诉别人，帮你带来新顾客
```

---

## 阶段 1：陌生人 → 好奇者（让他们知道你存在）

**这个阶段要完成的任务：**
让目标范围内的人知道「附近有这家店」，并且对它产生一点点兴趣。

**这个阶段用户的心理状态：**
「我从来没听说过这家店，不知道好不好，值不值得去。」

**有效的触达方式：**
- 小红书搜索流量（带地名+产品词的笔记）
- 大众点评搜索（搜「附近烘焙」时出现）
- 高德/百度地图（导航时出现）
- 口碑传播（朋友推荐是最有力的「第一次知道」）

**这个阶段的内容策略：**
- 内容要能回答：「这家店是做什么的？在哪里？值得去吗？」
- 不需要复杂的活动，需要的是：**清晰的产品展示 + 可信的地址/营业时间 + 一个「值得去一趟」的理由**
- 顾客口碑故事（哪怕是一条真实评价的截图）比任何广告都管用

**关键转化动作：**
从「知道了」到「有兴趣去」，需要一个「低风险的第一步」：
- 「先看看菜单/价格」（大众点评/小红书评论区）
- 「先问一下」（私信）
- 「路过顺便进去看看」（地图导航）

---

## 阶段 2：好奇者 → 初访者（让他们迈出第一步）

**这个阶段要完成的任务：**
把「有兴趣但没来」的人，变成真正来一次的人。

**这个阶段用户的心理状态：**
「我知道这家店，也有点兴趣，但我不确定值不值得专程去一趟。万一不好吃/不好停车/价格贵呢？」

**这个阶段的核心障碍：风险感**
人们不行动，是因为感觉「来一趟的成本（时间/钱/期望落空的风险）」大于「来了可能获得的好处」。

**降低第一次来的门槛（选1-2个用）：**

**方式 A：首次优惠（明确的利益钩子）**
「第一次来，说是从小红书看到的，给你一个小惊喜」（不要说具体是什么，制造好奇）
注意：「打折」是最弱的利益钩子，「送一个特别的东西」比「打折」更能驱动行动

**方式 B：社会证明（让他们看到「别人都觉得值」）**
- 真实顾客的小红书评价（不是刷的）
- 大众点评上的图文评价（有真实照片的）
- 顾客在群里/朋友圈自发分享的内容

**方式 C：减少决策风险（承诺机制）**
「不好吃你说，我给你换」——逆转风险，让人感觉「就算不好也没损失」

**方式 D：具体的来店理由（限时/限量/新品）**
「本周五新出一款[产品]，只做20个，想尝的要来」——给一个「现在去而不是以后去」的理由

---

## 阶段 3：初访者 → 常客（让第一次来变成习惯）

**这是整个生命周期里最关键的阶段。**
研究表明：本地服务类店铺，顾客第三次来以后，复购概率大幅提升。
**因此，让顾客从「来了一次」变成「再来两次」，是最重要的投资。**

**这个阶段用户的心理状态：**
「来过一次，感觉还不错。但我没有特别的理由一定要再来，除非路过或者想到了。」

**关键：主动制造「下次来的理由」**

在顾客第一次来的时候（不是之后，是当场），就要植入「回来的理由」。

**回来的理由设计（3种有效机制）：**

**机制 1：集章/集印卡（即时参与感）**
「买5次换一个你最喜欢的产品」
- 不要做积分App（学习成本太高），纸质集章卡就够
- 关键：在付款时当场给，而不是「下次来再给」

**机制 2：「下次来告诉你」的钩子（内容预告）**
「我们下周会有一个新口味，只有老顾客提前知道，你加一下我的微信」
- 不是推销，是给她一个「关注你」的理由
- 加了微信 = 进入私域 = 复购率大幅提升

**机制 3：个人化记忆（最高价值，最强连接）**
「你上次说你女儿喜欢草莓口味的，今天正好有新出的草莓款，给你留了一个」
- 这件事连锁店永远做不到
- 一次个人化的记忆，比所有优惠都更能让人回来

## 阶段 4：常客维护（防止流失）

**这个阶段要完成的任务：**
让已经有购买习惯的顾客，保持这个习惯，不要流失到竞争对手或者「忘记了这家店」。

**常客流失的最常见原因（不是不喜欢了，而是：）**
- 被遗忘：太久没被联系/提醒，「想不起来」就等于「没去」
- 新鲜感消失：产品一直没变化，没有「今天又有什么新的」的期待感
- 关系断裂：老板换了店员，或者服务质量波动，「不是那个感觉了」
- 竞争对手出现：附近新开了一家，好奇心驱动去试试，试了满意就留在那了

**常客维护的核心动作：**

**动作 1：保持存在感（不靠高频打扰，靠有价值的出现）**
- 微信群：每周 2-3 次内容（新品/日常/节日），不要超过3次（频率过高会被屏蔽）
- 不定期的「专属通知」：「你是我们的老顾客，明天限量产品提前告诉你」

**动作 2：制造期待感（让常客有「下次来」的理由）**
- 季节限定产品（草莓季/秋栗季/圣诞款），让人感觉「这个时节不来会错过」
- 不定期的「老顾客专属活动」（不是折扣，是特权感）
  例：「我们下周会出一款新产品，先给老顾客试吃，你想不想来？」

**动作 3：建立情感账户（日常积累）**
- 记得特殊日期（如果知道顾客生日/纪念日，提前一天发一条暖心消息）
- 在顾客评价/分享后，真诚地回应（不是模板回复，是真实说话）
- 节假日的简短问候（不附带任何促销，就是一句「你们国庆快乐」）

---

## 阶段 5：推荐者培育（让常客帮你带来新顾客）

**这个阶段要完成的任务：**
让已经很喜欢这家店的顾客，自然地把这家店推荐给身边的人。

**关键认知：推荐行为不是被奖励驱动的，是被情感驱动的**

人们推荐一家店，不是因为有返现，而是因为：
- 「我真的觉得这家很好，跟朋友说会让我显得会推荐东西」
- 「我跟朋友提起这家店，因为跟这家店有情感连接，说起来开心」
- 「朋友正好有需求，我刚好能帮上忙」

所以，培育推荐者的核心是：**让顾客与这家店的情感连接足够深，深到她主动想分享。**

**推荐机制设计（自然驱动，不是奖励驱动）：**

**机制 1：给顾客「值得分享」的体验**
- 让产品有可以拍的瞬间（出炉/拉丝/精美包装/意外惊喜）
- 给顾客一个「专属故事」：「你是我们第100个顾客，这个送你」

**机制 2：社交货币设计**
- 包装上印「[地名]手工烘焙，只在[具体地址]有」——让顾客分享时有清晰信息可转发
- 限量产品「告诉朋友也给她留一个」——分享的理由内嵌在产品机制里

**机制 3：显性激励（补充，不是核心）**
- 「带朋友来，你们各享优惠」（双向激励，比单向佣金更自然）
- 不要设计过于复杂的推荐机制（顾客懒得执行）
- 一句话能说清楚的机制才会被执行

## 生命周期管理的诊断清单

Agent 在帮老板分析问题时，用这个清单快速定位「顾客卡在了哪个阶段」：

| 老板说的问题 | 对应的阶段卡点 | 策略方向 |
|-----------|-------------|---------|
| 「没什么人来」 | 阶段1-2：知名度或第一次来的门槛 | 曝光+降低来店门槛 |
| 「来了一次但没再来」 | 阶段2-3：初访者转化为常客失败 | 即时建立「下次来的理由」 |
| 「感觉老顾客越来越少了」 | 阶段3-4：常客流失 | 存在感维护+新鲜感补充 |
| 「广告有人看但没人来」 | 阶段1-2：从知道到行动的转化 | 降低风险+具体来店理由 |
| 「口碑不错但增长慢」 | 阶段4-5：口碑没有被激活成推荐 | 设计推荐机制+社交货币 |

## 不同阶段的预算分配建议

```
如果月预算 1000 元，应该这样分配：
- 新店（阶段1-2为主）：70%用于曝光，30%用于降低来店门槛
- 成长期（阶段2-3为主）：50%用于带新客，50%用于初访者转化
- 稳定期（阶段3-5为主）：60%用于常客维护，40%用于口碑推荐激活
```

**关键原则：预算永远优先投在「比你落后一个阶段的顾客转化」上，而不是跳过阶段。**
例如：如果你缺常客，要先解决「初访者为什么没再来」，而不是去拉更多新陌生人进来。
